API changelog and releasesÂ¶

The default versioning system is MAJOR.MINOR.PATCH, where changes in the major number indicates that the API has been modified significantly, and the code examples or tests needed to be changed drastically to reflect that. Minor version changes usually add functionality and retain backwards compatibility in the same major number (mostly). Patches indicate backwards compatible bug fixes.

Additional labels for pre-release and build metadata are available as extensions to the MAJOR.MINOR.PATCH format.
